Overview

All endpoints use HTTP POST and require a Bearer token in the Authorization header. Requests and responses are JSON.

Before you begin Generate an access token using the Authentication endpoint. Include the token in every subsequent request as:
Authorization: Bearer <access_token>
Required Headers — All Endpoints
Header Value Required
Content-Type application/json Yes
Authorization Bearer <access_token> Yes (except Auth API)
01

Authentication API

Obtain an access token by submitting valid admin credentials. The token must be passed in the Authorization header of all subsequent requests.

POST
/V1/integration/admin/token
Request Body Parameters
ParameterTypeRequiredDescription
username string Yes Admin username registered for API access.
password string Yes Corresponding admin password.
Request Payload JSON 
{
  "username": "testapi",
  "password": "testapi123"
}
Response
200 OK "<access_token>"
The response is a plain string — the access token. Store it securely and pass it as Authorization: Bearer <token> in all subsequent requests.
02

Invoice Upload API

Submit invoice details for one or more items within an order. Each item requires an invoice number, date, and optional remarks.

POST
/V1/order/{orderId}/invoiceInfo
Path Parameters
ParameterTypeRequiredDescription
orderId integer Yes Unique identifier of the target order.
Request Body Parameters
ParameterTypeRequiredDescription
items array Yes Array of invoice item objects. Must contain at least one item.
items[].orderItemId string Yes Unique identifier of the order line item.
items[].invoiceNo string Yes Invoice number issued by the seller (e.g. INV-2024-001).
items[].invoiceDate string Yes Invoice date. Format: DD/MM/YYYY.
items[].invoiceRemarks string No Additional notes on the invoice. Pass "NA" if not applicable.
Request Payload JSON 
{
  "items": [
    {
      "orderItemId": "738",
      "invoiceNo":    "INV-2024-001",
      "invoiceDate":  "22/09/2024",
      "invoiceRemarks": "NA"
    }
  ]
}
Response
200 OK "<orderId>"
03

Shipment Upload API

Submit shipment details including item quantities, serial codes, and carrier tracking information for a specific order. Submitting this API automatically sets the order status to Packaged.

Automatic Status Transition On successful submission, the order status is automatically updated to Packaged. No separate status update call is required for this transition.
POST
/V1/order/{orderId}/shipment
Path Parameters
ParameterTypeRequiredDescription
orderId integer Yes Unique identifier of the target order.
Request Body — Root & Items
ParameterTypeRequiredDescription
dateTime string Yes Shipment dispatch date. Format: DD/MM/YYYY.
items array Yes Array of shipped item objects.
items[].orderItemId integer Yes Unique identifier of the order line item.
items[].qty integer Yes Quantity of the item being shipped.
items[].extensionAttributes.snCode string No Serial number (SN code) of the item, if applicable.
Request Body — Tracks (Carrier)
ParameterTypeRequiredDescription
tracks array Yes Array of carrier tracking objects.
tracks[].trackNumber string Yes Carrier-issued tracking number (e.g. TRK123456).
tracks[].title string Yes Display name of the carrier (e.g. FedEx, BlueDart).
tracks[].carrierCode string Yes System carrier code. Use "custom" for non-standard carriers.
Request Payload JSON 
{
  "dateTime": "22/09/2024",
  "items": [
    {
      "orderItemId": 738,
      "qty": 1,
      "extensionAttributes": {
        "snCode": "ABC123"
      }
    }
  ],
  "tracks": [
    {
      "trackNumber": "TRK123456",
      "title":       "FedEx",
      "carrierCode": "custom"
    }
  ]
}
Response
200 OK "<orderId>"
04

Status Upload API

Update the fulfilment status of specific order items. Supports the active order lifecycle from shipment through delivery and returns. Processing and Packaged statuses are system-managed and cannot be set via this API.

POST
/V1/order/{orderId}/orderStatus
Path Parameters
ParameterTypeRequiredDescription
orderId integer Yes Unique identifier of the target order.
Request Body Parameters
ParameterTypeRequiredDescription
items array Yes Array of status update objects.
items[].orderItemId string Yes Unique identifier of the order line item.
items[].orderStatus string Yes New status value. See Allowed Status Values below.
items[].dateTime string Yes Date of the status change. Format: DD/MM/YYYY.
items[].remarks string No Notes explaining the status change (e.g. reason for cancellation).
Request Payload JSON 
{
  "items": [
    {
      "orderItemId": "738",
      "orderStatus":  "Shipped",
      "dateTime":    "22/09/2024",
      "remarks":     "Dispatched via FedEx"
    }
  ]
}
Allowed Status Values
Shipped
Handed to carrier for delivery
Active
In Transit
In transit to delivery address
Active
Delivered
Successfully delivered to customer
Completed
Reverse Pick up Initiated
Return pickup request raised
Reverse
Return in Transit
Returned item on its way back
Reverse
Returned
Item received back at warehouse
Reverse
Response
200 OK "<orderId>"
05

Cancellation API

Cancel one or more items within an order. Use this endpoint to notify the system of customer-initiated or seller-initiated cancellations, along with the reason and timestamp.

POST
/V1/order/{orderId}/cancel
Important Cancellation is irreversible. Once an item is cancelled, its status cannot be reverted. Ensure all validations are performed on the client side before calling this endpoint.
Path Parameters
ParameterTypeRequiredDescription
orderId integer Yes Unique identifier of the target order.
Request Body Parameters
ParameterTypeRequiredDescription
items array Yes Array of cancellation objects. Must contain at least one item.
items[].orderItemId string Yes Unique identifier of the order line item to cancel.
items[].dateTime string Yes Date of cancellation. Format: DD/MM/YYYY.
items[].cancelReason string Yes Reason for cancellation (e.g. "Customer request", "Out of stock").
items[].remarks string No Additional notes regarding the cancellation.
Request Payload JSON 
{
  "items": [
    {
      "orderItemId":  "738",
      "dateTime":     "22/09/2024",
      "cancelReason": "Customer request",
      "remarks":      "Customer called to cancel before dispatch"
    }
  ]
}
Response
200 OK "<orderId>"
06

Refund Status Upload API

Update the refund processing status for returned order items. Use this endpoint to track refund progress from initiation through to completion or decline.

POST
/V1/order/{orderId}/refundStatus
Path Parameters
ParameterTypeRequiredDescription
orderId integer Yes Unique identifier of the target order.
Request Body Parameters
ParameterTypeRequiredDescription
items array Yes Array of refund status objects.
items[].orderItemId string Yes Unique identifier of the order line item.
items[].refundStatus string Yes Refund status value. See Allowed Refund Status Values below.
items[].dateTime string Yes Date of the refund status update. Format: DD/MM/YYYY.
items[].refundAmount number No Refund amount to be processed. Required when status is "Refund Initiated".
items[].remarks string No Notes on the refund decision or processing details.
Request Payload JSON 
{
  "items": [
    {
      "orderItemId":  "738",
      "refundStatus": "Refund Initiated",
      "dateTime":     "22/09/2024",
      "refundAmount": 1299.00,
      "remarks":      "Refund approved after quality inspection"
    }
  ]
}
Allowed Refund Status Values
Refund Pending
Refund request received, awaiting review
Refund
Refund Initiated
Refund approved and processing started
Refund
Refunded
Amount successfully credited to customer
Completed
Refund Declined
Refund request rejected after review
Declined
Response
200 OK "<orderId>"
⚠

Error Codes

All endpoints return standard HTTP status codes. Error responses include a message field with details.

HTTP Code Status Description
200 OK Request was successful.
400 Bad Request Invalid payload structure or missing required parameters.
401 Unauthorized Missing, expired, or invalid Bearer token.
404 Not Found The specified orderId or orderItemId does not exist.
422 Unprocessable Entity Business logic validation failed (e.g. invalid status value or date format).
500 Internal Server Error Unexpected server-side error. Contact the integration support team.
